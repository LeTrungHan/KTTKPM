Mô Tả Kiến Trúc

1. Các Thành Phần Chính

  • API Gateway: Điểm truy cập duy nhất cho client, routing requests đến các services tương ứng
  • Product Service: Quản lý thông tin sản phẩm (tên, giá, mô tả, tồn kho)
  • Order Service: Quản lý đơn hàng (tạo, xem, hủy đơn hàng)
  • Customer Service: Quản lý thông tin khách hàng (tên, địa chỉ, thông tin liên lạc)
  • Message Broker: RabbitMQ/Kafka cho giao tiếp bất đồng bộ giữa các services

2. Mô Hình Giao Tiếp

  • Đồng bộ: REST API/gRPC cho giao tiếp trực tiếp giữa các services
  • Bất đồng bộ: Event-driven communication qua Message Broker
  • Mẫu CQRS: Tách biệt thao tác đọc và ghi để tối ưu hóa hiệu suất
  • Xử lý sự kiện: Publish/Subscribe mô hình giao tiếp giữa các services

3. Lưu Trữ Dữ Liệu

  • Database per Service: Mỗi microservice có database MongoDB riêng
  • MongoDB Compass: Công cụ quản lý và trực quan hóa databases
  • Data Consistency: Sử dụng các mẫu như Saga để đảm bảo tính nhất quán của dữ liệu

4. Containerization

  • Docker: Mỗi service được containerized riêng biệt
  • Docker Compose: Định nghĩa và quản lý tất cả các containers như một ứng dụng thống nhất
  • Scalability: Dễ dàng scale các services độc lập với nhau
  • Portability: Đảm bảo ứng dụng chạy nhất quán trên mọi môi trường

Triển Khai Thực Tế

  1. Triển khai với Docker Compose: docker-compose up -d
  2. Mỗi service được build từ Dockerfile riêng, các Dockerfile được định nghĩa trong thư mục tương ứng
  3. MongoDB được khởi chạy trong container riêng, dữ liệu được persist thông qua Docker volumes
  4. MongoDB Compass kết nối đến các database containers qua các cổng được exposed
  5. API Gateway expose cổng duy nhất cho client, tất cả các requests đều đi qua Gateway
